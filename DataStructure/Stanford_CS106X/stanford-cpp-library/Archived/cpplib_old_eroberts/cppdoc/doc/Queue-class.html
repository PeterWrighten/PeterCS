class Queue<ValueType>

This class models a linear structure called a queue in which values are added at one end and removed from the other. This discipline gives rise to a first-in/first-out behavior (FIFO) that is the defining feature of queues. To provide additional flexibility, the Queue class also supports the concept of a priority queue in which elements are returned in priority order. As in conventional English usage, lower priority numbers correspond to higher effective priorities, so that a priority 1 item takes precedence over a priority 2 item.

Constructor

Queue() Initializes a new empty queue.

Methods

size() Returns the number of values in the queue.
isEmpty() Returns true if the queue contains no elements.
clear() Removes all elements from the queue.
enqueue(value, priority) Adds value to the queue with the specified priority.
dequeue() Removes and return the highest priority value.
peek() Returns the value of highest priority in the queue, without removing it.

Constructor detail

void enqueue(ValueType value, double priority = 0);
Adds value to the queue with the specified priority. Lower priority numbers correspond to higher priorities, which means that all priority 1 elements are dequeued before any priority 2 elements. If the priority argument is omitted, elements are added at priority 0.

Usage:

queue.enqueue(value, priority);

ValueType dequeue();
Removes and return the highest priority value. If multiple entries in the queue have the same priority, those values are dequeued in the same order in which they were enqueued.

Usage:

ValueType first = queue.dequeue();

ValueType peek();
Returns the value of highest priority in the queue, without removing it.

Usage:

ValueType first = queue.peek();
